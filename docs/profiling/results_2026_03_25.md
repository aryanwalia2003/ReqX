# Performance Profiling Report - March 25, 2026

## Executive Summary
A comprehensive performance audit was conducted on the ReqX CLI using the `go tool pprof` engine during a high-concurrency load test (up to 200 virtual users across ramping stages). The audit identified critical CPU hotspots in the network/security stack and major memory allocation bottlenecks within the JSON processing and I/O buffering layers.

---

## ⚡ CPU Performance Hotspots
| Component | CPU Usage | Root Cause Analysis |
| :--- | :--- | :--- |
| **Windows Syscalls** | ~39.2% | Heavy OS-level overhead for networking and file management on Windows. |
| **TLS Handshaking** | ~25.1% | Significant time spent in `crypto/tls.(*Conn).HandshakeContext`. Fresh handshakes for new VUs during scaling are extremely expensive. |
| **WebSocket Reading**| ~23.3% | Continuous stream processing overhead in `github.com/gorilla/websocket.(*Conn).ReadMessage`. |
| **JSON Decoding** | ~12.4% | CPU cycles spent in `encoding/json.(*decodeState).unmarshal` for dynamic response extraction. |

**Key Insight:** CPU usage is dominated by "Network Infrastructure" (TLS/Handshaking).
**Optimization:** Implementing more aggressive connection reuse or disabling certificate verification during internal load tests would provide the most immediate CPU relief.

---

## 🧠 Memory Allocation Bottlenecks
| Component | Allocation (MB) | Root Cause Analysis |
| :--- | :--- | :--- |
| **JSON interface{} Maps**| ~469.6 MB | Decoding response bodies into generic `interface{}` maps for variable extraction creates thousands of small, short-lived objects. |
| **I/O ReadAll Buffering**| ~282.6 MB | Full-body buffering in `io.ReadAll` and `io.Copy` before parsing. |
| **Socket.IO Channels** | ~214.5 MB | Background buffering of incoming real-time events in goroutine queues. |

**Key Insight:** Memory usage is dominated by "JSON Unmarshalling".
**Optimization:** Switching dynamic extraction to a streaming JSON parser (e.g., `segmentio/encoding/json`) would prevent loading entire response bodies into memory, significantly reducing the heap footprint.

---

## 🛠 Strategic Recommendations
1. **Streaming JSON Extraction:** Transition the `pm.env.set` and variable extraction logic to use a streaming approach instead of full-body `Unmarshal`.
2. **Buffer Segments:** Replace high-allocation `io.ReadAll` calls with pooled, segmented buffers to minimize the impact on the Garbage Collector (GC).
3. **Handshake Warm-up:** Use the `--stages` flag to warm up the connection pool gradually, reducing the peak CPU burst caused by thousands of simultaneous TLS handshakes.
